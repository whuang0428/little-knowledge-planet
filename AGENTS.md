# AGENT.md

## Project

This project is `little-knowledge-planet`, a Vite + React + Tailwind educational prototype for children.

The app presents small knowledge lessons, each with:

- a lesson card
- a short explanation
- a "动一动，看一看" interaction area
- a three-question quiz
- a parent discussion prompt
- a completion badge

The checked-in `README.md` is still the default React + Vite template. Treat the application code and data files as the real source of project behavior.

## Stack

- React 19
- Vite 8
- Tailwind CSS 4 through `@tailwindcss/vite`
- framer-motion for animation and drag/click motion
- lucide-react for UI icons
- JavaScript and JSX only; there is no TypeScript setup

## Important Files

- `src/App.jsx`: main application UI, routing state, lesson view, quiz behavior, and all interactive demos.
- `src/data/lessons.js`: lesson content, quiz data, interaction copy, badges, and parent prompts.
- `src/data/categories.js`: category ids, labels, icons, and category colors.
- `src/main.jsx`: React entrypoint.
- `src/index.css`: Tailwind import.
- `src/App.css`: leftover template CSS; it is not central to the current app.
- `vite.config.js`: Vite React plugin plus Tailwind plugin.
- `eslint.config.js`: ESLint flat config for JS/JSX.

Do not edit `node_modules` or `dist`. `dist` is generated by the build and is ignored by git.

## Commands

Use `npm.cmd` in PowerShell if `npm` is blocked by script execution policy.

- Install dependencies: `npm.cmd install`
- Start dev server: `npm.cmd run dev -- --host 127.0.0.1`
- Production build: `npm.cmd run build`
- Lint: `npm.cmd run lint`
- Preview built app: `npm.cmd run preview`

The Vite dev server is a long-running process. If it prints a local URL and keeps running, that is expected behavior, not a hang.

## Encoding

Project content includes Chinese text and emoji. Files should be treated as UTF-8.

If PowerShell displays Chinese as mojibake, do not "fix" the source text based on that terminal output. Use a UTF-8-aware read, browser view, or Node-based inspection before changing Chinese copy.

## App Structure

`ChildrenKnowledgeExplorerPrototype` in `src/App.jsx` owns the main app state:

- selected category
- active lesson
- current view
- quiz answers
- completed lesson ids
- moon phase
- reflection panel visibility

`InteractiveDemo` chooses special lesson demos by `lesson.id`. If no special demo matches, it falls back to `CategoryExploreDemo`.

Special demos currently include:

- `cat-eyes`
- `penguin-feet`
- `sunflower`
- `moon-shape`
- `rainbow`
- `pipa-string`

`CategoryExploreDemo` is the generic category template for all other lessons.

## Interaction Rules

The "动一动，看一看" section must not feel fake.

If visible copy says the user can click, drag, compare, or observe a change, the UI must visibly respond in that way. A valid response can be a state change, highlighted card, changed message, changed icon, movement, reveal, progress, or another obvious visual difference.

Keep interaction text and real behavior aligned:

- Do not use "点击" unless clicking changes something visible.
- Do not use "拖动" unless dragging changes something visible.
- Do not show a static visual while describing a changing experiment.
- Prefer small local React state for click-cycle demos.
- Prefer framer-motion for visible animation and drag feedback.
- Keep the existing soft, rounded, child-friendly visual style.

## Editing Guidance

Keep changes scoped to the active prompt.

When a prompt says to edit only one file, edit only that file. The previous Prompt 1 was explicitly limited to `App.jsx`; future prompts may have different limits.

Do not edit `src/data/lessons.js` or `src/data/categories.js` unless the prompt explicitly allows it.

Do not rewrite the app architecture for small interaction fixes. Prefer focused edits inside the existing component that owns the behavior.

For special demos, change only the relevant `lesson.id` branch inside `InteractiveDemo`.

For generic category behavior, change `CategoryExploreDemo`.

## Verification Checklist

After UI or interaction changes:

1. Run `npm.cmd run build`.
2. If lint-sensitive code was touched, run `npm.cmd run lint`.
3. Manually test the affected lesson in the browser.
4. For any click or drag prompt, verify that the UI visibly changes after the action.
5. For any copy change, verify that the displayed interaction text matches the real behavior.

When testing locally, use:

```powershell
npm.cmd run dev -- --host 127.0.0.1
```

Then open:

```text
http://127.0.0.1:5173/
```
